# November Trip — ระบบเว็บไซต์ทัวร์ + หลังบ้าน

เว็บไซต์ทัวร์ในประเทศไทย พร้อมระบบหลังบ้าน (admin) สำหรับจัดการทัวร์ การจอง ลูกค้า
และบทความ พร้อมส่งอีเมลยืนยันอัตโนมัติเมื่อมีการจอง

สร้างด้วย **Next.js 16 (App Router) + TypeScript + Tailwind CSS v4**, ใช้ **PostgreSQL**
เป็นฐานข้อมูล (ใช้ได้กับ Neon, Supabase, Railway, RDS หรือ Postgres ของตัวเอง) และ **Resend**
สำหรับส่งอีเมล — โครงสร้างนี้ **deploy บน Vercel ได้ทันทีโดยไม่ต้องแก้โค้ดเพิ่ม**

---

## ✨ ฟีเจอร์

**หน้าเว็บไซต์**
- หน้าแรกพร้อม hero แบบ carousel เปลี่ยนภาพอัตโนมัติ, แถบเมนูมีเมนูย่อย "จุดหมายปลายทาง"
- รายการทัวร์กรองตามหมวดหมู่/ประเทศ/กลุ่มเป้าหมาย (ครอบครัว/คู่รัก/กลุ่มเพื่อน/รักธรรมชาติ) พร้อมปุ่ม "โหลดเพิ่มเติม"
- หน้ารายละเอียดทัวร์แบบแท็บ (ภาพรวม / กำหนดการ / ที่พัก / ราคา) พร้อม breadcrumb, รหัสทัวร์,
  badge คะแนนรีวิว, ระดับความยาก/ความสบาย (dot scale), ตารางราคาตามขนาดกลุ่ม, รายการที่พักแต่ละคืน,
  ลิงก์วิดีโอ YouTube (ถ้ามี) และปุ่ม "ปรับแต่งทริปนี้" แยกจากแท็บ
- **สองโหมดการติดต่อในหน้าทัวร์**: ปุ่ม "จองทันที" กับ "ขอใบเสนอราคา" สลับได้ในฟอร์มเดียวกัน
- หน้า "ติดต่อเรา" มีสองแท็บ: ส่งข้อความทั่วไป และ **"ขอทัวร์ตามสั่ง 100%"** (แบบฟอร์มละเอียด: คำนำหน้า,
  WhatsApp, สัญชาติ, เดือน/ปีที่เดินทาง, จำนวนผู้ใหญ่/เด็ก/ทารก, งบประมาณ, ช่องทางที่รู้จักเรา, ความต้องการพิเศษ)
- หน้ารีวิวลูกค้า (`/reviews`) และส่วนรีวิวในหน้าแรก
- ปุ่มลอยติดต่อทีมงาน (WhatsApp / อีเมล / โทรศัพท์) แสดงทุกหน้า
- ฟุตเตอร์แสดงสถิติบริษัท (ปีประสบการณ์, จำนวนนักท่องเที่ยว ฯลฯ) และลิงก์รีวิวภายนอก — ตั้งค่าได้ในหลังบ้าน
- ส่วน "5 ขั้นตอนสู่ทริปในฝัน" อธิบายกระบวนการจองแบบ personalize

**ระบบหลังบ้าน (`/admin`)**
- ล็อกอินแบบมีรหัสผ่าน (bcrypt + session cookie)
- แดชบอร์ด: ยอดขายรายเดือน (กราฟ), จำนวนการจอง, จำนวนคำขอใบเสนอราคา, ลูกค้า, การจองล่าสุด
- จัดการทัวร์: เพิ่ม/แก้ไข/ลบ, รหัสทัวร์, วิดีโอ, กลุ่มเป้าหมาย (tags), ตารางราคาตามขนาดกลุ่ม,
  กำหนดการรายวัน (มื้ออาหาร/ที่พัก/รูปภาพ/เส้นทาง), คะแนนรีวิว, รายการที่พักหลายรายการต่อทริป,
  ข้อมูลเอเจนต์ประจำทริป, สถานะเผยแพร่
- จัดการการจอง: แยกประเภท "จองทันที" กับ "ขอใบเสนอราคา", เปลี่ยนสถานะ
- **จัดการคำขอทัวร์ตามสั่ง** (`/admin/trip-requests`): ดูรายละเอียดครบ พร้อมเปลี่ยนสถานะ
- จัดการลูกค้า, รีวิวลูกค้า (เพิ่ม/แก้ไข/ลบ), บทความ
- **ตั้งค่าเว็บไซต์** (`/admin/settings`): เบอร์ WhatsApp, อีเมล/เบอร์โทรติดต่อ, สถิติบริษัทในฟุตเตอร์,
  ลิงก์ TripAdvisor/Google Reviews/Facebook/Instagram

**อีเมลอัตโนมัติ**
- เมื่อมีการจอง: ส่งอีเมลยืนยันให้ลูกค้า + แจ้งเตือนแอดมิน
- เมื่อมีข้อความติดต่อ: ส่งอีเมลตอบรับอัตโนมัติ
- หากยังไม่ตั้งค่า `RESEND_API_KEY` ระบบจะ**ไม่ล่ม** — แค่บันทึกลง log แทนการส่งจริง
  (เหมาะสำหรับพัฒนา/ทดสอบก่อนต่ออีเมลจริง)

**การชำระเงินออนไลน์**: ยังไม่ได้ทำในเวอร์ชันนี้ (ตามที่แจ้งไว้ว่ายังไม่ต้องการตอนนี้) —
ลูกค้าจองแล้วชำระเงินนอกระบบ (โอน/QR/พบทีมงาน) ถ้าต้องการเพิ่มภายหลัง แจ้งได้เลยครับ
(แนะนำ Stripe สำหรับบัตรต่างประเทศ หรือ Omise/2C2P ถ้าต้องการรับ PromptPay)

**หมายเหตุเรื่องเนื้อหา**: โครงสร้างและฟีเจอร์ของเว็บนี้ได้รับแรงบันดาลใจจากเว็บทัวร์ต้นแบบที่ลูกค้าส่งมาให้ดู
แต่ **ชื่อแบรนด์ ชื่อพนักงาน รูปภาพ และรีวิวลูกค้าทั้งหมดเป็นข้อมูลตัวอย่าง** ที่สร้างขึ้นใหม่
ไม่ใช่ข้อมูลจริงของบริษัทต้นแบบ สามารถแก้ไขเปลี่ยนเป็นข้อมูลจริงของธุรกิจคุณได้ทั้งหมดผ่านหลังบ้านหรือแก้โค้ดโดยตรง

## 🎨 ปรับแต่งแบรนด์และรูปลักษณ์จากหลังบ้าน (ไม่ต้องแก้โค้ด)

ไปที่ **หลังบ้าน → ตั้งค่าเว็บไซต์** (`/admin/settings`) เพื่อแก้ไขได้เองทั้งหมด:

- **ชื่อเว็บไซต์** และ **ชื่อบริษัทเต็ม** — ใช้แสดงในเมนู ฟุตเตอร์ ชื่อหน้าเว็บ (title) ทุกจุด
- **โลโก้** — ค่าเริ่มต้นใช้ไฟล์ `public/logo.jpg` (โลโก้ที่ให้มา) เปลี่ยนได้โดยใส่ URL รูปใหม่ในหลังบ้าน
  หรือแทนที่ไฟล์ `public/logo.jpg` ในโค้ดโดยตรง (ระบบยังไม่มีระบบอัปโหลดไฟล์ผ่านหน้าเว็บ —
  ถ้าจะใช้ URL ภายนอก อัปโหลดไปที่บริการฝากรูปก่อน เช่น [Cloudinary](https://cloudinary.com),
  [imgbb.com](https://imgbb.com) แล้วคัดลอกลิงก์มาใส่)
- **โทนสีทั้งเว็บ** — เลือกสีพื้นหลัง สีตัวอักษร สีหลัก และสีเน้นด้วย color picker เปลี่ยนแล้วมีผลทันที
  ทั้งหน้าเว็บไซต์และหลังบ้าน (ค่าเริ่มต้นเป็น**โทนฟ้า-ขาว**ตามโลโก้: พื้นขาว + น้ำเงินกรมท่า + ฟ้าอมเทา)
- **รูปภาพหน้าแรก (Hero)** — ใส่ลิงก์รูปได้หลายรูป จะแสดงเป็นสไลด์เปลี่ยนอัตโนมัติ

การตั้งค่าเหล่านี้เก็บในฐานข้อมูล ไม่ใช่ในโค้ด จึงแก้ไขได้ตลอดเวลาโดยไม่ต้อง deploy ใหม่

## 💬 ป๊อปอัพที่ปรึกษา (มุมขวาล่าง)

ไอคอนลอยมุมขวาล่างของทุกหน้าใช้รูปที่ปรึกษาที่ตั้งไว้ในหลังบ้าน หลังจากลูกค้าเปิดเว็บไปสักครู่
(ประมาณ 3.5 วินาที) จะมีข้อความทักทายเด้งขึ้นมาอัตโนมัติ (แสดงครั้งเดียวต่อการเข้าเว็บหนึ่งครั้ง)
กดที่ข้อความหรือไอคอนเพื่อเปิดช่องทางติดต่อ (WhatsApp/อีเมล/โทร)

**แก้ไขได้ที่**: `/admin/settings` → หัวข้อ "ป๊อปอัพที่ปรึกษา" — เปลี่ยนชื่อ ตำแหน่ง รูปโปรไฟล์
และข้อความทักทายได้ทั้งหมด

## 🔗 วิดเจ็ต/ลิงก์พันธมิตร (เช่น GetYourGuide, Viator, Klook)

**แก้ไขได้ที่**: `/admin/partner-embeds`

วางโค้ด HTML ที่พันธมิตรให้มา (เช่นป้าย affiliate ของ GetYourGuide) ลงในช่องแล้วกดเพิ่ม —
ระบบจะแสดงในฟุตเตอร์ของทุกหน้าอัตโนมัติ เพิ่มได้หลายรายการพร้อมกัน ลบออกได้ตลอดเวลา

> โค้ด HTML ที่ใส่จะถูกแสดงผลตรงๆ (ไม่ผ่านการกรอง) เนื่องจากเป็นเนื้อหาที่แอดมินที่ล็อกอินแล้วเป็นคนใส่เอง
> จึงควรใส่เฉพาะโค้ดจากแหล่งที่เชื่อถือได้เท่านั้น

## 🖼️ อัปโหลดรูปภาพจริง (JPG/PNG/WEBP)

ทุกช่องใส่รูปภาพในหลังบ้าน (รูปหน้าปกทัวร์, รูปประจำวันในกำหนดการ, รูปที่พัก, รูปโปรไฟล์เอเจนต์/ที่ปรึกษา,
รูปหน้าปกบทความ, รูปโปรไฟล์รีวิว, โลโก้, รูปสไลด์หน้าแรก) **อัปโหลดไฟล์จากเครื่องได้โดยตรง**
ไม่ต้องหา URL จากที่อื่นมาใส่อีกต่อไป (แต่ยังวาง URL เองได้เหมือนเดิมถ้าต้องการ)

**วิธีใช้**: กดปุ่ม "อัปโหลดไฟล์ (JPG/PNG)" → เลือกไฟล์จากเครื่อง → รูปจะอัปโหลดและแสดงตัวอย่างทันที
→ กด "บันทึก" ของฟอร์มนั้นตามปกติเพื่อบันทึกลงฐานข้อมูล

**หลักการทำงาน**: รูปภาพที่อัปโหลดจะถูกเก็บลงในฐานข้อมูล PostgreSQL โดยตรง (ตาราง `uploads`)
และมี URL ภายในระบบ (เช่น `/api/uploads/12`) ให้ใช้อ้างอิงได้เหมือน URL รูปทั่วไปทุกจุดในเว็บ —
**ไม่ต้องสมัครบริการฝากไฟล์แยกต่างหาก** (เช่น S3, Cloudinary) ใช้งานได้ทันทีทั้งบน VPS และ Vercel

> ⚠️ **ข้อจำกัด**: จำกัดขนาดไฟล์ละ **5MB** และรองรับเฉพาะ JPG, PNG, WEBP — เนื่องจากรูปภาพเก็บอยู่ในฐานข้อมูล
> โดยตรง หากมีรูปภาพจำนวนมากหรือขนาดใหญ่ อาจทำให้ฐานข้อมูลโตเร็วและกระทบพื้นที่ใน free tier ของ
> Neon/Supabase ได้ — ถ้าต้องการระบบที่ scale ได้มากกว่านี้ในอนาคต แนะนำย้ายไปใช้ Vercel Blob หรือ
> Cloudinary แทน (แจ้งได้หากต้องการให้ช่วยทำ)

## 🎬 วิดีโอ YouTube ในหน้าทัวร์

ช่อง "วิดีโอ YouTube (URL)" ในหน้าแก้ไขทัวร์ — วางลิงก์ YouTube (ทั้งแบบ `youtube.com/watch?v=...`
หรือ `youtu.be/...`) จะแสดงเป็น**วิดีโอฝังในหน้าเว็บ**ที่เล่นได้ทันที (ไม่ใช่แค่ลิงก์ให้กดออกไปดู)
ในแท็บ "ภาพรวม" ของหน้ารายละเอียดทัวร์

## 🗺️ แผนที่เส้นทาง + ไทม์ไลน์การเดินทาง

**อยู่ที่**: หน้ารายละเอียดทัวร์ → แท็บ **"กำหนดการ"** (Itinerary)

- **แผนที่**: ใส่ลิงก์ Google Maps Embed ได้ในหน้าแก้ไขทัวร์ (`/admin/tours/[id]/edit` →
  ช่อง "แผนที่เส้นทาง") จะแสดงแผนที่จริงไว้ด้านบนสุดของแท็บกำหนดการ
  วิธีหาลิงก์: เปิด Google Maps ค้นหาสถานที่/เส้นทาง → กด "แชร์" → แท็บ "ฝังแผนที่" (Embed a map)
  → คัดลอกเฉพาะ URL ที่อยู่ใน `src="..."` — **ไม่ต้องสมัคร API key หรือเสียค่าใช้จ่ายใดๆ**
- **ไทม์ไลน์เส้นทางรายวัน**: ในหน้าแก้ไขทัวร์ แต่ละวันของกำหนดการสามารถเพิ่ม "ช่วงเดินทาง" ได้
  หลายช่วง (เช่น กรุงเทพฯ → บางปะอิน → อยุธยา → กรุงเทพฯ) แต่ละช่วงระบุ ต้นทาง/ปลายทาง/เวลา/
  ระยะทาง/พาหนะได้ ระบบจะแสดงเป็นเส้นทางแนวตั้งพร้อมไอคอนพาหนะ (รถยนต์/เรือ/เครื่องบิน/เดินเท้า/รถไฟ
  — เลือกไอคอนให้อัตโนมัติจากคำที่พิมพ์ในช่อง "พาหนะ") พร้อมรูปภาพประกอบของวันนั้นๆ

ทั้งสองส่วนบันทึกลงฐานข้อมูลและแก้ไขได้ตลอดเวลาจากหลังบ้าน ไม่ต้องแก้โค้ด

## 💱 เลือกสกุลเงินได้ (THB, USD, EUR, GBP, JPY, AUD, CAD)

ลูกค้าเลือกสกุลเงินที่ต้องการดูได้จากมุมขวาบน (ข้างปุ่มเลือกภาษา) — ราคาทุกจุดในเว็บ
(หน้ารายการทัวร์, หน้ารายละเอียด, ฟอร์มจอง) จะแปลงจากบาทเป็นสกุลเงินที่เลือกให้อัตโนมัติ

**สกุลเงินจะสลับให้ตรงกับภาษาที่เลือกโดยอัตโนมัติ**:
ไทย → THB, English → USD, Français → EUR, Deutsch → EUR, 日本語 → JPY
ถ้าลูกค้าเลือกสกุลเงินเองในภาษาใดภาษาหนึ่ง ระบบจะจดจำไว้เฉพาะภาษานั้น (เก็บแยกตามภาษาในเบราว์เซอร์)
พอสลับกลับไปภาษาอื่นจะกลับไปใช้สกุลเงินที่เหมาะกับภาษานั้นตามปกติ

**ตั้งค่าอัตราแลกเปลี่ยนได้ที่หลังบ้าน** (`/admin/settings`):
- กดปุ่ม **"ดึงอัตราแลกเปลี่ยนล่าสุด"** เพื่อดึงอัตราจริงจากอินเทอร์เน็ตอัตโนมัติ (ใช้ API ฟรีไม่ต้องมี key)
- หรือกรอกอัตราแลกเปลี่ยนเองในแต่ละช่องได้ตลอดเวลา

> หมายเหตุ: ราคาทัวร์ทุกรายการยังคงกรอกเป็น**เงินบาทเสมอ**ในหน้าจัดการทัวร์ — ระบบจะแปลงเป็น
> สกุลเงินอื่นให้ตอนแสดงผลเท่านั้น ไม่กระทบข้อมูลราคาต้นฉบับ

ใช้งานได้ตั้งแต่จอมือถือ, แท็บเล็ต, ไปจนถึงจอคอมพิวเตอร์ขนาดใหญ่ — เมนู, ฟอร์มจอง, การ์ดทัวร์,
ตารางข้อมูล (เลื่อนแนวนอนได้บนจอเล็ก), และฟอร์มในหลังบ้าน ปรับเลย์เอาต์ตามขนาดหน้าจออัตโนมัติ

**หลังบ้านบนมือถือ**: มีปุ่มเมนูแฮมเบอร์เกอร์ (☰) ที่มุมซ้ายบนของทุกหน้าหลังบ้านเมื่อเปิดจากมือถือ/แท็บเล็ต
กดเพื่อเปิดเมนูเลื่อนออกด้านข้าง เข้าถึงทุกเมนูหลังบ้านได้เหมือนบนจอคอมพิวเตอร์

## 🚀 เริ่มต้นใช้งาน (Local)

**ข้อกำหนด:** Node.js 20.9 ขึ้นไป และฐานข้อมูล PostgreSQL (ใช้ฟรีจาก [neon.tech](https://neon.tech)
หรือ [supabase.com](https://supabase.com) ก็ได้ ไม่จำเป็นต้องติดตั้ง Postgres ในเครื่องตัวเอง)

```bash
npm install
cp .env.example .env.local     # แล้วใส่ DATABASE_URL และแก้ค่าอื่นๆ ตามต้องการ
npm run dev
```

เปิด http://localhost:3000 — ตารางฐานข้อมูลและผู้ดูแลระบบคนแรกจะถูกสร้างอัตโนมัติในการรันครั้งแรก
(ไม่ต้องรันคำสั่ง migrate เอง)

**เข้าสู่ระบบหลังบ้านครั้งแรก** ที่ `/admin/login` ด้วยค่าเริ่มต้น:
- อีเมล: ค่าใน `ADMIN_EMAIL` (ค่าเริ่มต้น `admin@example.com`)
- รหัสผ่าน: ค่าใน `ADMIN_PASSWORD` (ค่าเริ่มต้น `ChangeMe123!`)

> ⚠️ เปลี่ยนค่าเหล่านี้ใน `.env.local` **ก่อน**รันครั้งแรกในเครื่องจริง/production เพราะระบบจะสร้าง
> ผู้ดูแลระบบจากค่านี้แค่ครั้งเดียวตอนฐานข้อมูลว่างเปล่า

**ลืมรหัสผ่าน หรือบัญชีแอดมินเข้าไม่ได้เลย?** รีเซ็ตรหัสผ่านตรงกับฐานข้อมูลได้โดยไม่ต้องผ่านหน้าเว็บ:
```bash
npm run reset-admin-password -- admin@example.com "รหัสผ่านใหม่อย่างน้อย8ตัว"
```
สคริปต์นี้ใช้การเข้ารหัสแบบเดียวกับที่แอปใช้จริง (bcrypt) รับประกันว่าเข้าสู่ระบบด้วยรหัสใหม่ได้ทันที ต้องตั้งค่า `DATABASE_URL` ไว้ใน `.env.local` ก่อน (ไฟล์เดียวกับที่แอปใช้)

## 🔧 Environment Variables

ดูตัวอย่างทั้งหมดใน `.env.example`

| ตัวแปร | จำเป็น | คำอธิบาย |
|---|---|---|
| `DATABASE_URL` | **จำเป็น** | Connection string ของ PostgreSQL เช่น `postgresql://user:pass@host:5432/db` |
| `PGSSL` | ไม่บังคับ | ตั้งเป็น `disable` เฉพาะกรณีรัน Postgres ในเครื่องตัวเองแบบไม่มี SSL — ปล่อยว่างไว้สำหรับ Neon/Supabase/RDS |
| `ADMIN_EMAIL` | แนะนำ | อีเมลผู้ดูแลระบบที่จะถูกสร้างอัตโนมัติครั้งแรก |
| `ADMIN_PASSWORD` | แนะนำ | รหัสผ่านผู้ดูแลระบบเริ่มต้น |
| `SESSION_SECRET` | **จำเป็นสำหรับ production** | คีย์เซ็น session cookie — ตั้งเป็นสตริงสุ่มยาวๆ |
| `RESEND_API_KEY` | ไม่บังคับ | คีย์จาก [resend.com](https://resend.com) เพื่อส่งอีเมลจริง — ไม่ใส่ก็รันได้ปกติ (แค่ไม่ส่งอีเมลจริง) |
| `EMAIL_FROM` | ไม่บังคับ | อีเมลผู้ส่ง เช่น `Tours <no-reply@yourdomain.com>` (ต้อง verify โดเมนกับ Resend ก่อนใช้โดเมนตัวเอง) |
| `ADMIN_NOTIFY_EMAIL` | ไม่บังคับ | อีเมลที่จะรับการแจ้งเตือนเมื่อมีการจอง/ข้อความใหม่ |

### การตั้งค่า Resend (ส่งอีเมลจริง)
1. สมัครที่ https://resend.com (มี free tier)
2. สร้าง API key แล้วใส่ใน `RESEND_API_KEY`
3. (แนะนำ) เพิ่มโดเมนของคุณใน Resend และ verify DNS records เพื่อส่งจากอีเมลโดเมนตัวเอง
   แล้วตั้ง `EMAIL_FROM=Tours <no-reply@yourdomain.com>`
4. ตั้ง `ADMIN_NOTIFY_EMAIL` เป็นอีเมลที่ต้องการรับแจ้งเตือนการจองใหม่

## ✅ พร้อม Deploy บน Vercel ได้ทันที

โปรเจกต์นี้ใช้ **PostgreSQL** เป็นฐานข้อมูลหลักตั้งแต่ต้น (ไม่ใช่ SQLite) จึงไม่มีข้อจำกัดเรื่อง
disk ถาวรแบบที่ serverless มักเจอ — **deploy บน Vercel ได้เลยโดยไม่ต้องแก้โค้ดเพิ่ม** แค่มี
Postgres connection string (จาก Neon, Supabase ฯลฯ) ก็พอ

> 💡 บน Vercel/serverless แนะนำใช้ connection string แบบ **pooled** ของผู้ให้บริการ
> (เช่น Neon จะมี host ที่ลงท้ายด้วย `-pooler`) เพื่อป้องกันปัญหาเปิด connection เกินจำนวนที่รองรับ
> เวลามีหลาย function ทำงานพร้อมกัน

## 📦 Deploy

📖 **ดูคู่มือ deploy แบบละเอียดขั้นตอนต่อขั้นตอนได้ที่ [`DEPLOY.md`](./DEPLOY.md)**
(ทั้งแบบ Vercel และแบบ VPS พร้อม Node.js, PM2, Nginx, SSL ฟรีด้วย Let's Encrypt)

สรุปแบบย่อ:

### แบบ Vercel (แนะนำ — เร็วและง่ายที่สุด)
1. สมัคร Postgres ฟรีที่ [neon.tech](https://neon.tech) หรือ [supabase.com](https://supabase.com)
2. Push โค้ดขึ้น GitHub
3. Import โปรเจกต์เข้า [vercel.com](https://vercel.com)
4. ตั้งค่า Environment Variables ทั้งหมด (`DATABASE_URL`, `ADMIN_EMAIL`, `ADMIN_PASSWORD`,
   `SESSION_SECRET`, `RESEND_API_KEY` ฯลฯ) ในหน้า Settings ของ Vercel
5. Deploy — ตารางฐานข้อมูลและแอดมินคนแรกจะถูกสร้างอัตโนมัติตอนเปิดเว็บครั้งแรก

### แบบ VPS ของตัวเอง
```bash
npm install
npm run build
NODE_ENV=production npm run start
```
แนะนำใช้ `pm2` ให้รันตลอดเวลา และตั้ง reverse proxy (Nginx) หน้าเว็บ — รายละเอียดครบใน `DEPLOY.md`

## 🌐 ระบบหลายภาษา (i18n)

เว็บไซต์ฝั่งลูกค้ารองรับ **5 ภาษา** ผ่าน next-intl พร้อมใช้งานครบ 100% ของข้อความในเว็บ:
ไทย, English, Français (ฝรั่งเศส), Deutsch (เยอรมัน), 日本語 (ญี่ปุ่น)

ทุกภาษามี URL prefix ของตัวเอง (`/th/tours`, `/en/tours`, `/fr/tours` ฯลฯ) เพื่อให้หน้าแรกสุด (`/`)
ว่างไว้เป็น **หน้าเลือกภาษา** (language landing page) — ผู้เข้าชมครั้งแรกจะเห็นหน้าให้เลือกภาษาก่อน
เข้าสู่เว็บไซต์ สลับภาษาระหว่างท่องเว็บได้จากปุ่มเลือกภาษาที่แถบเมนูบนสุด (มุมขวา) — ถ้ามีคนเข้าลิงก์
แบบไม่มี prefix (เช่น `/tours`) ระบบจะ redirect ไปยังภาษาเริ่มต้นให้อัตโนมัติ — ระบบหลังบ้าน (`/admin`)
ไม่ได้แปลภาษาเพราะเป็นเครื่องมือใช้งานภายใน

### โครงสร้างไฟล์ที่เกี่ยวข้อง
```
app/page.tsx        หน้าเลือกภาษา (แสดงที่ URL "/" ก่อนเข้าสู่เว็บไซต์)
i18n/
  routing.ts        รายชื่อภาษาที่รองรับ + ภาษาเริ่มต้น (แก้ที่นี่เพื่อเพิ่ม/ลบภาษา)
  navigation.ts      Link/useRouter/usePathname แบบรู้จักภาษาปัจจุบันอัตโนมัติ
  request.ts         โหลดไฟล์คำแปลตามภาษาที่ใช้งาน
messages/
  th.json            คำแปลภาษาไทย
  en.json            คำแปลภาษาอังกฤษ
proxy.ts             (เดิมชื่อ middleware.ts) จัดการ routing ตามภาษา ยกเว้นเส้นทาง /admin
app/[locale]/        หน้าเว็บไซต์สาธารณะทั้งหมดอยู่ใต้โฟลเดอร์นี้
```

### วิธีเพิ่มภาษาใหม่ (เช่น ภาษาจีน — นอกเหนือจาก 5 ภาษาที่มีอยู่แล้ว)
1. เปิด `i18n/routing.ts` เพิ่มโค้ดภาษาใน `locales`, ชื่อภาษาใน `localeLabels`,
   และรูปภาพประกอบใน `localeImages` (แสดงในหน้าเลือกภาษา)
   ```ts
   export const locales = ["th", "en", "fr", "de", "ja", "zh"] as const;
   // ...
   export const localeLabels = { /* ...ภาษาเดิม... */ zh: "中文" };
   export const localeImages = { /* ...ภาษาเดิม... */ zh: "https://..." };
   ```
2. คัดลอก `messages/en.json` เป็น `messages/zh.json` แล้วแปลค่าทุกตัว (**ห้ามเปลี่ยนชื่อ key**)
3. เสร็จแล้ว — เมนูเลือกภาษาและระบบ routing จะรองรับภาษาใหม่ให้อัตโนมัติ ไม่ต้องแก้โค้ดหน้าอื่น

### วิธีเพิ่มข้อความใหม่ในหน้าที่มีอยู่
1. เพิ่ม key ใหม่ในไฟล์ `messages/th.json` **และ** `messages/en.json` (โครงสร้าง key ต้องตรงกันทั้งสองไฟล์)
2. ในไฟล์หน้าเว็บ (server component) เรียกใช้ผ่าน `getTranslations("ชื่อ namespace")`
   หรือใน component ฝั่ง client เรียกผ่าน `useTranslations("ชื่อ namespace")` จาก `next-intl`

### เนื้อหาที่แอดมินกรอกเอง (ชื่อทัวร์, รายละเอียด, ป๊อปอัพที่ปรึกษา ฯลฯ)
ตอนนี้แปลได้แล้ว ไม่ใช่แค่ข้อความ UI ของระบบ:

- **ทัวร์แต่ละรายการ**: ในหน้าแอดมิน "จัดการทัวร์ → แก้ไข" มีส่วน **"คำแปลภาษาอื่น"** ท้ายฟอร์ม
  ให้กรอกคำแปลแยกทีละภาษา (EN/FR/DE/JA) สำหรับชื่อทัวร์, หมวดหมู่, จังหวัด, คำโปรย, รายละเอียด,
  จุดเด่น, ราคารวม/ไม่รวม, ข้อมูลที่พัก, ตำแหน่งเอเจนต์ และกำหนดการรายวัน — ถ้าไม่กรอกช่องไหน
  หน้าเว็บภาษานั้นจะ fallback ไปแสดงภาษาไทยให้อัตโนมัติ ไม่มีวันขึ้นเป็นช่องว่าง
- **ป๊อปอัพที่ปรึกษา**: ในหน้าแอดมิน "ตั้งค่า" มีช่องแปลตำแหน่ง/ข้อความทักทายแยกตามภาษาเช่นกัน
- **ปุ่ม "แปลร่างอัตโนมัติ (ฟรี)"**: ทั้งสองจุดข้างต้นมีปุ่มนี้ต่อภาษา ดึงคำแปลร่างจาก
  [MyMemory](https://mymemory.translated.net) (บริการแปลภาษาฟรี ไม่ต้องมี API key) มาใส่ให้ก่อน
  แล้วแอดมินตรวจ/แก้คำก่อนกดบันทึกจริง — เป็นแค่ตัวช่วยร่าง ไม่ใช่คำแปลที่รับประกันความถูกต้อง
  100% และมีโควต้าฟรีจำกัด (~5,000 คำ/วันต่อ IP) ถ้าแปลจำนวนมากในคราวเดียวอาจแปลไม่ครบทุกช่อง —
  กดปุ่มซ้ำหรือกรอกเองในช่องที่เหลือได้เสมอ
- หมวดหมู่/จังหวัด ที่ใช้เป็นตัวกรองหน้ารายการทัวร์ก็แปลได้ในลักษณะเดียวกัน (การกรองจริงยังอิงค่า
  ภาษาไทยที่เก็บในฐานข้อมูลเป็นหลัก ผู้ใช้เห็นแค่ป้ายที่แปลแล้วเท่านั้น)
- **บทความ (News) และรีวิวลูกค้า** ยังไม่ได้ต่อระบบแปลนี้ในตอนนี้ — ใช้สถาปัตยกรรมเดียวกันได้
  ถ้าต้องการเพิ่มในอนาคต

### ข้อจำกัดปัจจุบัน (รู้ไว้ก่อนใช้งานจริง)
- ข้อความ**แจ้งเตือนจากฟอร์ม** (เช่น ข้อความ validation ตอนกรอกฟอร์มผิด) ในหน้าแอดมินยังเป็นภาษาไทยเท่านั้น
  (เป็นเครื่องมือหลังบ้านสำหรับพนักงาน ไม่ใช่หน้าที่ลูกค้าเห็น) ส่วนข้อความฝั่งลูกค้าแปลแล้วครบทุกภาษา

## 🗂 โครงสร้างโปรเจกต์

```
app/
  [locale]/               หน้าเว็บไซต์สาธารณะทั้งหมด (รองรับหลายภาษา — ดูหัวข้อ i18n ด้านบน)
  admin/(dashboard)/      หน้าหลังบ้านที่ต้องล็อกอิน (แดชบอร์ด, ทัวร์, การจอง, ลูกค้า, บทความ ฯลฯ)
  admin/login/            หน้าล็อกอิน (ไม่ต้องล็อกอินก่อนเข้า)
i18n/                     การตั้งค่าภาษา (routing, navigation helpers)
messages/                 ไฟล์คำแปล th.json / en.json
components/               React components (แบ่ง public / admin)
lib/
  db.ts                   การเชื่อมต่อ PostgreSQL + schema/migrations (สร้างตารางอัตโนมัติตอนรันครั้งแรก)
  queries/                ฟังก์ชันอ่าน/เขียนข้อมูลทั้งหมด (ทัวร์, การจอง, บทความ ฯลฯ) — ทุกฟังก์ชันเป็น async
  actions/                Server Actions (ฟอร์มจอง, ล็อกอิน, จัดการข้อมูลหลังบ้าน)
  auth.ts                 ระบบล็อกอินผู้ดูแล (bcrypt + JWT cookie)
  email.ts                ส่งอีเมลผ่าน Resend
  seed.ts                 ข้อมูลตัวอย่าง (ทัวร์ 6 รายการ + บทความ 2 บทความ + รีวิว 3 รายการ)
proxy.ts                  จัดการ routing ตามภาษา (เดิมชื่อ middleware.ts)
```
